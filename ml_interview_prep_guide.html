ML Foundations

Everything from the bias-variance tradeoff to model evaluation. Master these cold.

Bias-Variance Tradeoff HOT
Core
▶

The expected prediction error decomposes into three parts:

E[(y - ŷ)²] = Bias² + Variance + Irreducible Noise Bias = E[ŷ] - y (systematic error) Variance = E[(ŷ - E[ŷ])²] (sensitivity to training data)
ScenarioBiasVarianceFix
UnderfittingHighLowMore features, complex model
OverfittingLowHighRegularize, more data, dropout
Just rightLowLow—
💡 Interview tip: Always mention that adding more training data reduces variance but not bias. Increasing model complexity trades bias for variance.

Diagnosing via learning curves

  • High bias: both train and val error are high and converge
  • High variance: large gap between train (low) and val (high) error
  • Good fit: train ≈ val, both acceptably low
Regularization — L1, L2, Elastic Net
Core
▶
L2 (Ridge): Loss = MSE + λ·Σwᵢ² → shrinks all weights, never zero L1 (Lasso): Loss = MSE + λ·Σ|wᵢ| → sparse weights, feature selection Elastic Net: Loss = MSE + λ₁·Σ|wᵢ| + λ₂·Σwᵢ² → best of both
MethodSparsityHandles correlated featuresUse when
L2 RidgeNoYes (groups)All features relevant
L1 LassoYesPicks one, drops restFeature selection needed
Elastic NetPartialYesMany features, some correlated

Why L1 induces sparsity — geometric intuition

The L1 ball is a diamond shape with corners on the axes. Contours of the loss function are likely to touch these corners first, forcing some weights to exactly zero. L2's circular ball has no corners, so optimal points are rarely exactly zero.

Model Evaluation & Metrics HOT
Core
▶

Classification metrics

Precision = TP / (TP + FP) → of predicted positives, how many are correct? Recall = TP / (TP + FN) → of actual positives, how many did we find? F1 = 2 · (P·R) / (P+R) → harmonic mean, balances precision & recall AUC-ROC = P(score_pos > score_neg) → threshold-invariant ranking quality
⚠️ Use AUC-PR (Precision-Recall curve) for imbalanced datasets, not AUC-ROC — ROC is overly optimistic when negatives vastly outnumber positives.

When to use which metric

Use caseMetricWhy
Fraud detectionRecall + AUC-PRMissing fraud (FN) is costly
Spam filterPrecisionBlocking legit email (FP) is costly
Medical screeningRecall / SensitivityMissing disease is costly
Ranking (search)NDCG, MRR, MAPPosition matters
RegressionRMSE, MAE, R²RMSE penalizes outliers more

Cross-validation

  • k-fold CV: Split into k folds, train on k-1, validate on 1. Rotate. Average results.
  • Stratified k-fold: Maintains class distribution per fold. Use for imbalanced data.
  • Leave-one-out (LOO): k = n. Expensive but low bias. For small datasets.
  • Time series split: Only use past data to predict future. Never shuffle time series!
Feature Engineering & Selection
Concept
▶

Encoding categorical variables

  • One-hot encoding: Low-cardinality features. Creates n binary columns.
  • Label encoding: Ordinal features only (Small=0, Med=1, Large=2).
  • Target encoding: Replace category with mean target value. Risk: data leakage — do within CV folds.
  • Embeddings: High-cardinality categories in neural networks.

Handling missing values

  • MCAR: Missing completely at random → simple imputation ok
  • MAR: Missing at random (depends on other features) → model-based imputation
  • MNAR: Missing not at random → needs domain knowledge, may be informative

Feature selection methods

TypeMethodPros
FilterCorrelation, mutual information, χ²Fast, no model needed
WrapperRFE, forward/backward selectionModel-specific, better
EmbeddedL1 Lasso, tree feature importanceEfficient, integrated

Scaling

  • StandardScaler: Zero mean, unit variance. Use for SVM, linear models, PCA.
  • MinMaxScaler: [0,1] range. Use for neural networks, when distribution unknown.
  • RobustScaler: Uses IQR. Use when outliers present.
  • Log transform: For right-skewed distributions (income, count data).

Math
▶

Learning rate schedules

  • Step decay: Reduce by factor every N epochs
  • Cosine annealing: Smooth decay following cosine curve
  • Warmup + cosine: Standard for transformer training
  • Cyclical LR: Smith 2017 — cycles between min and max lr
💡 Saddle points are far more common than local minima in high-dimensional spaces. Gradient descent escapes saddle points with noise (mini-batch) and momentum.
Class Imbalance — Handling Skewed Data
Concept
▶

Data-level methods

  • SMOTE: Synthetic Minority Over-sampling — interpolates between minority samples in feature space
  • Random oversampling: Duplicate minority samples (risk: overfitting)
  • Random undersampling: Remove majority samples (risk: information loss)
  • Tomek links: Remove majority samples near decision boundary

Algorithm-level methods

  • Class weights: class_weight='balanced' in sklearn — inversely proportional to frequency
  • Threshold tuning: Move decision threshold from 0.5 based on cost matrix
  • Focal loss: Down-weights easy examples, focuses on hard ones (used in RetinaNet)
Focal Loss = -αₜ(1-pₜ)^γ · log(pₜ) where γ=2 typical

ML Algorithms

Every algorithm you need to know deeply — math, intuition, pros/cons, and implementation.

Linear & Logistic Regression
Algorithm
▶

Linear Regression

ŷ = Xw + b Closed form: w* = (XᵀX)⁻¹Xᵀy (O(n·d²) — only for small d) Gradient: ∇L = (2/n)Xᵀ(Xw - y)

Logistic Regression

σ(z) = 1/(1+e⁻ᶻ) (sigmoid) P(y=1|x) = σ(wᵀx + b) Loss = -[y·log(ŷ) + (1-y)·log(1-ŷ)] (Binary Cross-Entropy) Gradient: ∂L/∂w = (1/n)Xᵀ(ŷ - y) (elegant — same form as linear!)

Assumptions of linear regression

  • Linearity: E[y|x] is linear in x
  • Independence: residuals are independent
  • Homoscedasticity: constant residual variance
  • Normality: residuals ~ N(0, σ²) [for inference, not prediction]
  • No multicollinearity (check VIF < 10)
from sklearn.linear_model import LogisticRegression, Ridge
model = LogisticRegression(C=1.0, penalty='l2', solver='lbfgs')
model.fit(X_train, y_train)
proba = model.predict_proba(X_test)[:, 1]
Support Vector Machines (SVM)
Algorithm
▶

Finds the maximum-margin hyperplane separating classes. Only support vectors (points closest to boundary) determine the hyperplane.

Primal: min ½||w||² subject to yᵢ(wᵀxᵢ + b) ≥ 1 Dual: max Σαᵢ - ½ΣΣαᵢαⱼyᵢyⱼxᵢᵀxⱼ s.t. Σαᵢyᵢ = 0, 0 ≤ αᵢ ≤ C Kernel trick: K(xᵢ, xⱼ) = φ(xᵢ)ᵀφ(xⱼ) (no explicit φ needed)
KernelFormulaUse when
LinearxᵀzHigh-dim, linearly separable
RBF/Gaussianexp(-γ||x-z||²)General purpose
Polynomial(γxᵀz + r)^dImage features
Sigmoidtanh(γxᵀz + r)Neural net-like
⚠️ SVM scales as O(n²) to O(n³). Not suitable for >100k samples. Use SGDClassifier with hinge loss instead.
Decision Trees, Random Forests & Boosting HOT
Algorithm
▶

Decision tree splitting criteria

Gini: G = 1 - Σpᵢ² (default in sklearn) Entropy/IG: H = -Σpᵢ·log₂(pᵢ) IG(S, A) = H(S) - Σ(|Sᵥ|/|S|)·H(Sᵥ) Variance reduction (regression): Var(parent) - weighted Var(children)

Random Forest — why it works

  • Bagging: bootstrap samples → decorrelates trees
  • Random feature subsets: sqrt(d) features at each split → further decorrelation
  • Averaging: reduces variance without increasing bias
  • OOB error: ~37% of samples left out per tree → free validation set

Clustering — K-Means, DBSCAN, GMM
Algorithm
▶

K-Means

Minimizes within-cluster sum of squares (WCSS = Σᵢ Σₓ∈Cᵢ ||x - μᵢ||²). E-step: assign points to nearest centroid. M-step: update centroids. Converges to local optimum — run multiple times with k-means++.

K-Means++: Choose centroids with probability ∝ D(x)² → Better initialization, faster convergence, better solutions

DBSCAN

Density-based. Parameters: ε (neighborhood radius), MinPts (min neighbors). Core point: ≥MinPts in ε-ball. Border point: in ε-ball of core. Noise: neither.

✓ Finds arbitrary-shaped clusters, handles outliers (noise), no need to specify k. Fails in varying densities (use HDBSCAN).

Gaussian Mixture Models (GMM)

p(x) = Σₖ πₖ · N(x; μₖ, Σₖ) where Σπₖ = 1 EM algorithm: E-step: rᵢₖ = πₖN(xᵢ|μₖ,Σₖ) / Σⱼ πⱼN(xᵢ|μⱼ,Σⱼ) M-step: μₖ = ΣᵢrᵢₖΧᵢ / Σᵢrᵢₖ

Clustering evaluation (no labels)

  • Silhouette score: (b-a)/max(a,b) — higher is better. Range [-1,1]
  • Davies-Bouldin: Lower is better. Ratio of within-to-between cluster distances
  • Calinski-Harabasz: Higher is better. Variance ratio criterion
Dimensionality Reduction — PCA, t-SNE, UMAP
Math
▶

PCA — Principal Component Analysis

1. Center data: X̃ = X - μ 2. Covariance: Σ = X̃ᵀX̃ / (n-1) 3. Eigendecomposition: Σ = VΛVᵀ 4. Project: Z = X̃V_k (keep top-k eigenvectors) Equivalently via SVD: X = UΣVᵀ → principal components = columns of V
  • PCA finds directions of maximum variance
  • Components are orthogonal (uncorrelated)
  • Choose k by explained variance ratio (e.g., 95%)
  • Standardize before PCA when features have different scales

t-SNE vs UMAP (visualization)

t-SNEUMAP
SpeedSlow O(n²)Fast O(n log n)
PreservesLocal structureLocal + global
DeterministicNoYes (with seed)
Use forSmall datasets, vizLarge datasets, viz + downstream
⚠️ Neither t-SNE nor UMAP preserve distances — cluster sizes and inter-cluster distances are NOT meaningful. Only topology/neighborhood matters.
Recommendation Systems HOT
Algorithm
▶

Collaborative Filtering

User-based: find similar users, recommend what they liked. Item-based: find similar items to what user liked.

Matrix Factorization: R ≈ PQᵀ (P: user embeddings, Q: item embeddings) Loss: min Σ(rᵢⱼ - pᵢᵀqⱼ)² + λ(||P||² + ||Q||²) SGD update: pᵢ ← pᵢ + η(eᵢⱼqⱼ - λpᵢ) where eᵢⱼ = rᵢⱼ - pᵢᵀqⱼ

Two-tower architecture (Amazon/Google)

  • User tower: encodes user features, history → user embedding
  • Item tower: encodes item features → item embedding
  • Score: cosine similarity or dot product
  • Train with in-batch negatives or sampled negatives
  • Serve: ANN search (FAISS, ScaNN) for fast retrieval

Cold start problem solutions

  • Content-based: use item/user features when no interactions exist
  • Popularity-based: recommend trending items to new users
  • Hybrid: blend collaborative + content-based
  • Exploration: epsilon-greedy, UCB, or Thompson sampling (bandit approach)
Naive Bayes, KNN, and Bayesian Classifiers
Algorithm
▶

Naive Bayes

P(y|x) ∝ P(y) · Π P(xᵢ|y) (conditional independence assumption) Gaussian NB: P(xᵢ|y) = N(xᵢ; μᵢᵧ, σᵢᵧ²) Multinomial NB: P(xᵢ|y) = (count + α) / (total + α·|V|) [Laplace smoothing]

K-Nearest Neighbors

Non-parametric. Store all training data. Predict by majority vote (classification) or mean (regression) of k nearest neighbors.

  • k=1: very high variance. k=n: very high bias (predict mode)
  • Use cross-validation to find optimal k
  • Distance metrics: Euclidean, Manhattan, Cosine, Hamming (binary)
  • Scale features first! Distance is sensitive to scale.
  • O(nd) prediction — use KD-tree or Ball-tree for speedup

Deep Learning

Neural networks, transformers, LLMs, CNNs, and modern architectures.

Backpropagation & Automatic Differentiation HOT
Math
▶

Backprop is just the chain rule applied to a computational graph, computing gradients from output to input.

Forward: z = Wx + b; a = σ(z) Backward: ∂L/∂W = ∂L/∂a · ∂a/∂z · ∂z/∂W = δ · xᵀ where δ = ∂L/∂z = ∂L/∂a · σ'(z) Chain rule: ∂L/∂xₗ = Wₗ₊₁ᵀ · δₗ₊₁ (propagate error backward)

Vanishing & exploding gradients

  • Vanishing: Gradients → 0 as layers increase (sigmoid/tanh saturate). Fix: ReLU, residual connections, batch norm, LSTM gates
  • Exploding: Gradients grow exponentially. Fix: gradient clipping (by norm or value)
Gradient clipping: if ||g|| > threshold: g ← g · threshold/||g||

Transformers & Self-Attention HOT
Architecture
▶

Scaled dot-product attention

Attention(Q, K, V) = softmax(QKᵀ / √dₖ) · V Q = XWQ, K = XWK, V = XWV (linear projections) √dₖ scaling: prevents softmax from entering saturation region Multi-Head: head_i = Attention(QWᵢQ, KWᵢK, VWᵢV) MHA = Concat(head₁,...,headₕ) · WO

Why attention works

  • Every token attends to every other token → captures long-range dependencies
  • Attention weights are input-dependent (unlike convolution)
  • Parallelizable over sequence (unlike RNNs)
  • O(n²d) complexity — bottleneck for long sequences

Positional encoding

PE(pos, 2i) = sin(pos / 10000^(2i/d_model)) PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model)) Learned PE (BERT), RoPE (LLaMA), ALiBi, NoPE (various LLMs)

Efficient attention variants

  • FlashAttention: IO-aware, fused CUDA kernel, O(n) memory (vs O(n²))
  • Sparse attention: Local + global tokens (Longformer, BigBird)
  • Linear attention: O(n) — kernel approximation of softmax
  • GQA (Grouped Query Attention): Multiple query heads share key/value heads — used in LLaMA-3, Gemini

Layer Normalization

LayerNorm(x) = γ · (x - μ) / (σ + ε) + β (per-token, across features) Pre-LN (norm before attention) vs Post-LN (norm after) — Pre-LN more stable training
Large Language Models — Training & Fine-tuning HOT
Architecture
▶

Pre-training objectives

  • Causal LM (GPT): Predict next token. Loss = CE(p(xₜ|x₁..xₜ₋₁), xₜ)
  • Masked LM (BERT): Predict masked tokens. 15% masked: 80% [MASK], 10% random, 10% unchanged
  • Seq2Seq (T5): Encoder-decoder, span corruption denoising

RLHF — Reinforcement Learning from Human Feedback

1. SFT: finetune on demonstrations 2. Reward model: r(x,y) ← human preference pairs (Bradley-Terry model) 3. PPO: max_{π} E[r(x,y)] - β·KL(π||π_SFT) (KL prevents reward hacking)

  • KV Cache: Cache K,V tensors for past tokens during generation
  • Speculative decoding: Draft model generates tokens, target model verifies in parallel

Normalization, Dropout & Regularization in DL
Concept
▶
Batch Norm: μ_B = mean(xᵢ), σ²_B = var(xᵢ) x̂ᵢ = (xᵢ - μ_B) / √(σ²_B + ε) yᵢ = γx̂ᵢ + β (learnable scale/shift) Layer Norm: normalize across features (not batch) → used in Transformers Group Norm: normalize within channel groups → CNNs with small batch size

Why Batch Norm works

  • Reduces internal covariate shift (controversial — Santurkar et al. 2018 disputes this)
  • Empirically: smooths the loss landscape, allows higher learning rates
  • Acts as regularizer (slight noise from mini-batch statistics)
  • Use layer norm in transformers (batch size varies; sequence positions independent)

Dropout

Training: mask = Bernoulli(p); x̃ = x * mask / (1-p) [inverted dropout] Test: use all neurons (no scaling needed with inverted dropout)
  • p=0.1-0.3 for transformers, p=0.5 for dense layers
  • DropPath / Stochastic depth: drop entire residual blocks
  • Not used after conv layers much — BatchNorm subsumes regularization effect
Generative Models — VAE, GAN, Diffusion NEW
Architecture
▶

VAE — Variational Autoencoder

ELBO = E_q[log p(x|z)] - KL(q(z|x) || p(z)) = reconstruction loss + KL regularization Reparameterization: z = μ + ε·σ, ε~N(0,I) → allows backprop through sampling

GAN — Generative Adversarial Network

min_G max_D E_x[log D(x)] + E_z[log(1 - D(G(z)))] Issues: Mode collapse, training instability, vanishing gradients Fixes: WGAN (Wasserstein distance), spectral norm, gradient penalty

Diffusion Models (DDPM, DDIM)

Forward: q(xₜ|xₜ₋₁) = N(xₜ; √(1-β)xₜ₋₁, βI) [add noise] Reverse: p_θ(xₜ₋₁|xₜ) = N(xₜ₋₁; μ_θ(xₜ,t), Σ_θ) [denoise] Loss: L = E[||ε - ε_θ(xₜ, t)||²] [predict added noise]
💡 Diffusion models outperform GANs on image quality and diversity. DDIM allows ~10x faster sampling with deterministic inversion.

  • LLN: Sample mean → population mean as n→∞. Why more data helps.
Hypothesis Testing & A/B Testing HOT
Concept
▶

Framework

  • H₀ (null): No effect exists
  • H₁ (alternative): Effect exists
  • p-value: P(data as extreme | H₀ true) — NOT P(H₀ true|data)!
  • α (significance level): Acceptable false positive rate (typ. 0.05)
  • Power (1-β): P(reject H₀ | H₁ true). Typ. 0.80
Type I error (α): FP — reject H₀ when H₀ true Type II error (β): FN — fail to reject H₀ when H₁ true Effect size (Cohen's d) = (μ₁ - μ₂) / σ_pooled Sample size: n ≈ 2σ²(z_α/2 + z_β)² / δ²

A/B testing at scale (Amazon/Microsoft style)

  • Randomization unit: User-level (not page-level) — network effects, SUTVA
  • Novelty effect: New features get artificial boost. Run until novelty wears off.
  • Multiple testing: Bonferroni or BH correction when testing many metrics
  • Metric choice: North star + guardrail metrics. Don't optimize proxy too hard.
  • CUPED: Controlled experiment using pre-experiment data to reduce variance by 50-80%
  • Switchback tests: For marketplace/two-sided platforms where user-level randomization breaks
from scipy import stats
# Two-sample t-test
t_stat, p_val = stats.ttest_ind(control, treatment)
# With CUPED variance reduction
y_cuped = y - theta * (x_pre - x_pre.mean())
theta = np.cov(y, x_pre)[0,1] / np.var(x_pre)

Math

MLE vs MAP

MLE: θ* = argmax P(X|θ) (no prior) MAP: θ* = argmax P(X|θ)·P(θ) = MLE + prior regularization L2 prior ↔ L2 regularization (Ridge) L1 prior ↔ L1 regularization (Lasso)
Causal Inference HOT
Concept
▶

Correlation ≠ Causation. Causal inference asks: what is the effect of treatment T on outcome Y?

Potential outcomes framework (Rubin)

ATE = E[Y(1) - Y(0)] (average treatment effect) ATT = E[Y(1) - Y(0) | T=1] (effect on treated) Fundamental problem: cannot observe both Y(1) and Y(0) for same unit

Methods

  • RCT / A-B test: Gold standard. Random assignment ensures ignorability.
  • Propensity score matching: Match treated/control units with similar P(T=1|X)
  • Diff-in-diff (DiD): Compare changes over time: (treated-after) - (treated-before) - (control-after) + (control-before)
  • Instrumental variables (IV): Use instrument Z that affects T but not Y directly
  • Regression discontinuity (RD): Treatment assigned by threshold. Compare just above/below cutoff.
  • Synthetic control: Construct counterfactual from weighted combination of control units
⚠️ Confounders, selection bias, and SUTVA violations are the main pitfalls. Always ask: why is someone in treatment vs control?

-- Window functions (very common in interviews)
SELECT user_id, event_date, revenue,
  ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY event_date) as rn,
  SUM(revenue) OVER (PARTITION BY user_id ORDER BY event_date 
                     ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) as cum_rev,
  LAG(revenue, 1) OVER (PARTITION BY user_id ORDER BY event_date) as prev_rev,
  AVG(revenue) OVER (PARTITION BY user_id 
                     ORDER BY event_date ROWS 6 PRECEDING) as rolling_7d_avg
FROM events;

-- Cohort analysis (retention)
WITH first_order AS (
  SELECT user_id, MIN(order_date) as cohort_date FROM orders GROUP BY user_id
),
cohort AS (
  SELECT f.user_id, 
         DATEDIFF(month, f.cohort_date, o.order_date) as months_since
  FROM first_order f JOIN orders o USING(user_id)
)
SELECT cohort_date, months_since, COUNT(DISTINCT user_id) as retained
FROM cohort GROUP BY 1, 2;

-- Self join for session gap detection
SELECT a.user_id, a.ts, b.ts as next_ts,
       DATEDIFF(minute, a.ts, b.ts) as gap
FROM events a
LEFT JOIN events b ON a.user_id = b.user_id 
  AND b.ts = (SELECT MIN(ts) FROM events 
              WHERE user_id = a.user_id AND ts > a.ts);

MLOps — Production ML
System
▶

Model deployment patterns

  • Shadow deployment: New model runs alongside old, no serving traffic
  • Canary deployment: Route 1-5% traffic to new model before full rollout
  • Blue-green: Switch all traffic at once with instant rollback
  • Multi-armed bandit: Dynamically allocate traffic based on performance

Data/model drift

  • Data drift: P(X) changes — monitor feature distributions (KS test, PSI)
  • Concept drift: P(Y|X) changes — monitor model performance metrics
  • Label drift: P(Y) changes — monitor prediction distribution

Feature store

Centralized repository for features. Offline store (batch, historical) + online store (low-latency, point-in-time). Prevents training-serving skew. Examples: Feast, Tecton, AWS SageMaker Feature Store.
